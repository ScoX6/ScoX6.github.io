或者你可以直接使用Homebrew进行安装

brew cask install iterm2

二、配置Oh My Zsh

Oh My Zsh是对主题进一步扩展，下载地址：https://github.com/robbyrussell/oh-my-zsh

命令行安装：

sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

安装好之后，需要把Zsh设置为当前用户默认Shell

chsh -s /bin/zsh

iOS集成FFmpeg

###一、Mac配置FFmpeg环境

  1. 安装homebrew

    打开终端输入以下命令行：

`ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

`

  1. 安装FFmpeg

    1）利用上面的homebrew安装FFmpeg：

    brew install ffmpeg

    2）当命令结束之后，输入以下命令(查看你的安装ffmpeg的信息)：

    brew info ffmpeg

    详细请参考：Mac 配置FFmpeg环境

二、iOS集成FFmpeg

  1. 下载脚本FFmpeg脚本地址

  1. 编译FFmpeg-iOS-build-script，获得FFmpeg静态库文件

    打开脚本文件，然后使用终端运行：
    ./build-ffmpeg.sh

    错误1：

    fix:

    sudo xcode-select --switch /Applications/Xcode.app

  2. iOS项目集成FFmpeg

    1) 上述操作执行成功后，会生成FFmpeg-iOS文件夹，将该文件直接拖到项目中

    2) 配置头文件搜索路径：在工程文件 -> Build Setting -> Search Paths -> Header Search Paths 添加 $(SRCROOT)/$(PRODUCT_NAME)/FFmpeg-iOS/include (请根据实际路径更改)

4) 编译后会报错，然后根据提示挨个修复，还需要导入相应依赖的库

四、使用FFmpeg转码

  1. FFmpeg使用命令行调用

    1) 如ffmpeg -i /temp.mp3 -y /test.aac 这条指令就是调用ffmpeg,输入源(-i)是/temp.mp3文件,输出到/test.aac,其中-y参数是若存在则直接覆盖

    2) ffmpeg -i /temp.mp4 -f flv -y /temp.flv 这条指令是输入一个视频文件,将文件转码为h264编码格式的flv文件,-f参数是指定目标格式

    3) 具体FFmpeg能使用哪些指令可以参考官方文档或如下博客:http://www.cnblogs.com/wainiwann/p/4128154.html

    这里举一个通过FFmpeg将视频转码的例子。

  1. 获取转码进度条

    1) 如果想调用C里面函数需要写一个桥接类，创建任意一个Cocoa Touch Class，点h文件中所有东西都删除掉，点m中留下头文件,点h 中可声明c函数 在点m中调用 就能完成C和OC之间的通信

    2) 监控开始和完成状态,在C++函数中调用OC方法,来传递状态信息,结束线程前调用stopRuning()结束方法

    3) 获取视频文件总时间长度,ffmpeg_opt.c的open_input_file方法中会有时长信息ic->duration,为long long int类型数据

    4) 定时获取当前进度时间,ffmpeg.c的print_report方法中会输出Log,从log中获取当前的进度信息,为char info[1024]类型数据

    5) 制作进度条,转换百分比为当前进度除以总时长。注意事项:更改进度条的时候,是在非主线程,所以无法更改UI,需要在主线程执行更改UI操作

AOPAspect思路分析

Posted on 2019-12-12

AOPAspect 思路分析

源码

AOPAspect.h
AOPAspect.m

思路

  • 绑定思路

    1. 提取目标方法(Method)的实现(IMP)
    2. 替换目标方法(Method)的实现为_objc_msgForward
    3. 自身生成一个forwardingTargetForSelector：的映射方法并绑定forwardingTargetForSelector:实现(IMP)
    4. 替换目标的forwardingTargetForSelector：的实现(IMP)为自身的baseClassForwardingTargetForSelector：实现(IMP)
    5. 自身生成目标方法的一个映射方法，并绑定步骤1的实现(IMP)
    6. 存储为替换类型
    7. 存储为外部绑定类型

  • 路由思路

    1. 恢复目标方法的默认实现
    2. 循环存储结构，并调用内部绑定的block
    3. 重新绑定
